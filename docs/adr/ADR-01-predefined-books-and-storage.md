# [ADR-01] 事前定義作品の提供、デコード方式およびしおり保持仕様

## 日付
2026-06-12

## ステータス
承認済 (Accepted)

## コンテキスト
ゆうぞら（Yuzora）プロジェクトにおいて、以下の4つの主要な技術的・デザイン的な決定を行う必要がありました。

1. **スマートフォンのUX向上**: スマートフォンなどでローカルファイルのダウンロード・ファイル選択操作を行うのは不便であるため、事前定義されたマスターデータ作品（吉川英治「宮本武蔵」8作品）をワンタップで直接読み込めるようにしたい。
2. **マスターデータ作品の表示UI**: プレミアムな読書体験を提供するビューアーとして、ウェルカム画面での作品選択肢を美しく提示したい。
3. **文字コードの自動デコードとフォールバック**: ユーザーが外部からファイルを読み込む際、Shift_JIS（CP932）と UTF-8 が混在するため、自動判定してデコードする必要がある。
4. **しおり機能とセッション復元**: 複数ファイルの進捗を保持しつつ、アプリ再起動時に一瞬で直前のファイルを復元して読書を再開できる軽量なストレージ設計を行いたい。

## 意思決定

### 1. 事前定義作品（マスターデータ）の配置およびロード
* 吉川英治「宮本武蔵」8作品について、あらかじめShift_JIS形式の青空文庫テキストファイルをUTF-8へデコードしたファイルをリポジトリの `/src/books/` 配下に格納します。
* ウェルカム画面で作品が選択された際、`fetch` API を用いて対応する静的テキストファイルを非同期で取得し、パースおよび描画に回します。

### 2. 事前定義作品の選択UIデザイン
* 単なるドロップダウン等の無機質なUIではなく、ウェルカム画面上に日本の伝統的な書籍カバー（縦書きのタイトル帯など）を模した和風のグリッド状の「作品カード（選択ボタン）」を配置します。
* 各カードにはホバー時に滑らかに拡大・浮き上がるアニメーション（CSS Transitions）を適用し、プレミアムなUXを提供します。

### 3. 文字コード判定およびフォールバック
* `TextDecoder` API を利用し、`new TextDecoder('shift-jis', { fatal: true })` でまずデコードを実行します。
* デコード中に不正なバイトシーケンス（UTF-8等）を検知して例外（Error）がスローされた場合、それを `catch` して自動的に `new TextDecoder('utf-8')` によるデコードへフォールバックします。

### 4. 状態保持（しおり）とセッション復元の管理設計
* **最後の読書セッション（直近1ファイル）**: 
  * アプリ起動時に即時復元するため、テキスト本文データそのもの、ファイル名、ファイル形式を `localStorage` に保持します（`last_read_file_content`, `last_read_file_name`, `last_read_file_type`）。
* **過去の読書履歴（複数ファイル対応）**: 
  * 過去に読んだファイルすべての進捗率（スクロール位置パーセンテージ）のみを `bookmark_ファイル名` というキーで `localStorage` に永続化（実数値文字列、例: `0.4578`）します。
  * 同じファイルを再度読み込む（またはマスターから選択する）と、その進捗率を取得してビューアーのスクロール位置を自動復元します。

## 結果 (Consequences)

### メリット
* スマホやタブレット等のドラッグ＆ドロップが不便なデバイスにおいて、ファイル操作なしで即座に宮本武蔵などの小説を読み始められるため、UXが劇的に向上します。
* 完全なクライアントサイド静的SPAでありながら、前回の読書位置の復元や設定の永続化、複数本の進捗管理をローカル環境のみで軽量に実現できます。
* 自動フォールバックデコードにより、文字コードに詳しくないユーザーでも文字化けに悩まされることがなくなります。

### デメリット / 注意点
* 事前定義作品を増やす場合、手動またはスクリプトで `/src/books/` 内に UTF-8 のテキストファイルを事前に配置しておく必要があります（本プロジェクトでは初期必須8作品に限定しているため影響は最小限）。
