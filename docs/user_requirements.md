# 要求定義書 (User Requirements Definition) - こいぞら (Koizora)

## 1. はじめに (Introduction)
### 1.1 背景 (Background)
青空文庫（ https://www.aozora.gr.jp/ ）には多くの名作小説（吉川英治「宮本武蔵」など）が公開されているが、Webブラウザでのデフォルト表示（XHTML版など）は横書きであり、小説を日本古来の縦書きで読みたい読者にとっては読みづらさを感じる課題がある。

### 1.2 目的 (Objectives)
本プロジェクト「こいぞら（Koizora）」は、青空文庫のテキストおよびHTMLファイルをドラッグ＆ドロップするだけで、美しい縦書き表示に変換し、ブラウザ上で本物の本のように快適に読める静的ウェブビューアーを提供する。

### 1.3 アーキテクチャドメインとの位置づけ (TOGAF EA Business Architecture)
本ドキュメント（要求定義書）は、こいぞらプロジェクトにおける **TOGAF EA（エンタープライズアーキテクチャ）のビジネスアーキテクチャ (BA)** の定義に相当します。ユーザーの本質的な読書プロセスや、本物の書籍に近い縦書き読書体験というビジネス価値を整理し、動作保証の対象やターゲットユーザーを規定します。

## 2. ターゲットユーザーおよび想定ユースケース (Target Users & Use Cases)
* **ターゲットユーザー**: 
  * 青空文庫で名作小説を縦書きで楽しみたい小説愛好家。
  * 電子書籍リーダーを購入せず、PCやスマートフォンのブラウザだけで手軽に縦書き読書をしたいユーザー。
* **主要ユースケース**:
  1. 青空文庫からダウンロードした小説のテキスト（`.txt`）またはXHTML（`.html`）をブラウザへドラッグ＆ドロップし、縦書き表示で読む。
  2. スマートフォンやタブレットなどで、画面上に表示された事前定義済みの作品リスト（マスターデータ）からタイトルを選択するだけで、ローカルファイルのダウンロードやファイル選択の操作を行うことなく、ワンタップで縦書きの読書を開始する。
  3. 読書途中でブラウザを閉じ、後で同じ本を開いた際に前回の位置（しおり）から読書を再開する（最初のページから本を開いた際は、確実に1ページ目から開始されること）。
  4. 部屋の明るさに合わせてテーマ（配色）を変更したり、文字サイズ・行間を調整して自分好みの表示に変えて読む。
  5. 縦書き小説の進行方向について、伝統的な「右から左（RTL）」のみでなく、現代的な「左から右（LTR）」へオプションで切り替えて読む。また、最初のページへ戻るボタンの表示（ダブルアローの向き）を読書方向（RTLなら右向き「>>」、LTRなら左向き「<<」）に動的に連動させて違和感なく操作する。
  6. 長大な作品において、最初のページ（先頭）へ戻るボタンや、任意のページ番号の指定入力、進捗状況バー上の目印（つまみ）をドラッグ（またはクリック）して直感的にページを切り替えることで、読みたいページへ瞬時に移動して読む。進捗状況バーの進行方向およびつまみの移動方向は、読書方向（RTLなら右から左へ進行、LTRなら左から右へ進行）と一致させて自然な方向感覚を提供する。
  7. 読書中に表示された上部メニュー（ヘッダー）や下部情報（フッター）を、本文表示領域をタップ（またはクリック）することで即座に非表示にし、読書に集中できるようにする。また、スマートフォンやタブレットでの横スワイプ操作時には、画面がすべりすぎず正確に「1ページ分だけ」めくられ、かつめくり動作の方向（RTLなら右スワイプで次ページ、左スワイプで前ページ；LTRなら左スワイプで次ページ、右スワイプで前ページ）を読書方向に適合させて快適に読む。

* **モバイル・レイアウトおよび配置の制約**:
  * スマートフォンなどの画面幅の狭い端末において、文章が複数段（二段など）に分割されて表示されるのを防ぎ、 **常に単一カラム（1段組み）** で表示すること。
  * 縦書きとしての自然な読みやすさを確保するため、文章の表示開始位置（縦位置）を **常に上側に揃えて（上寄せ）表示** すること。

## 4. 今後の検討事項 (Future Roadmap)
* **複数ファイルの管理 / 本棚機能**: 過去に読み込んだファイルの履歴管理やブックマークの一覧表示。
* **目次ナビゲーション**: 長編小説における章ごとのジャンプ機能。
* **ルビ・注記の網羅性向上**: より複雑な青空文庫の外字注記や挿絵画像への対応。
