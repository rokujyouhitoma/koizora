# 要求定義書 (Requirement Definition Document) - コイゾラ (Koizora)

## 1. はじめに (Introduction)
### 1.1 背景 (Background)
青空文庫（https://www.aozora.gr.jp/）には多くの名作小説（吉川英治「宮本武蔵」など）が公開されているが、Webブラウザでのデフォルト表示（XHTML版など）は横書きであり、小説を日本古来の縦書きで読みたい読者にとっては読みづらさを感じる課題がある。

### 1.2 目的 (Objectives)
本プロジェクト「コイゾラ（Koizora）」は、青空文庫のテキストおよびHTMLファイルをドラッグ＆ドロップするだけで、美しい縦書き表示に変換し、ブラウザ上で本物の本のように快適に読める静的ウェブビューアーを提供する。

---

## 2. システム概要 (System Overview)
* **構成**: HTML5、Vanilla CSS、Pure JavaScriptのみで構成されるシングルページアプリケーション（SPA）。
* **動作環境**: 主要なモダンブラウザ（Chrome, Safari, Edge, Firefox等）。スマートフォンおよびPC双方に対応（レスポンシブ）。
* **対応フォーマット**:
  * 青空文庫テキストファイル（`.txt`）
  * 青空文庫XHTML/HTMLファイル（`.html`, `.xhtml`）
* **入力方法**: ドラッグ＆ドロップ、およびファイル選択ダイアログ（ファイル選択UI）によるファイルの読み込み。
* **文字コード自動判定/デコード**:
  * 青空文庫標準の Shift_JIS (CP932) デコードを第一優先。
  * デコード失敗時は UTF-8 へのフォールバック。

## 3. 機能要件 (Functional Requirements)

### 3.1 対象作品（初期対応必須作品）
本システムが正常かつ美しく表示できることを保証する最低限必須の動作確認対象として、以下の吉川英治「宮本武蔵」8作品（青空文庫収録）に完全対応すること。
* 宮本武蔵 01 序、はしがき（作品ID：52395）
* 宮本武蔵 02 地の巻（作品ID：52396）
* 宮本武蔵 03 水の巻（作品ID：52397）
* 宮本武蔵 04 火の巻（作品ID：52398）
* 宮本武蔵 05 風の巻（作品ID：52399）
* 宮本武蔵 06 空の巻（作品ID：52400）
* 宮本武蔵 07 二天の巻（作品ID：52401）
* 宮本武蔵 08 円明の巻（作品ID：52402）

### 3.2 青空文庫記法パース機能
* **ルビ（ふりがな）変換**:
  * 境界記号あり（例：`｜漢字《かんじ》` または `|漢字《かんじ》`）のパース。
  * 境界記号なし（例：連なる漢字に直接続く `漢字《かんじ》`）のパース。
* **改ページ処理**: `［＃改ページ］` を検知し、マルチカラム（ページ）の区切りとして適切にレンダリング（改ページ）。
* **傍点（強調マーク）対応**: `［＃「...」に傍点］` などの注記を検知し、傍点表示クラス（`.bouten`）を適用し、文字の右側に傍点を付与して表示。
* **不要注記のクレンジング**: 青空文庫特有のレイアウト指示注記（`［＃ここから...］`等）や制御文字を自動除去し、本文テキストのみをクリーンに表示。

### 3.3 読書画面（ビューアー）機能
* **縦書き表示**: `writing-mode: vertical-rl` を用いた日本語本来の右から左へ流れる本格的な縦書きレイアウトでの表示。
* **ページ送りナビゲーション**:
  * 画面左右エリアのクリック/タップによるページめくり（RTLのため、左側のクリック/タップで次のページへ進み、右側のクリック/タップで前のページへ戻る）。
  * キーボードの矢印キー（左キーで次のページへ進み、右キーで前のページへ戻る）による操作。
* **ヘッダー/フッターUI**:
  * **ヘッダー**: 戻るボタン、本タイトル、設定ボタンを表示。読書中はユーザー操作の停止（マウス不活動/タップ）を検知して自動非表示（3秒）。
  * **フッター**: 読書進捗率（%）および現在のページ番号（現在ページ / 総ページ数）を表示。
  * **プログレスバー**: 視覚的な進捗状況バー。

### 3.4 表示カスタマイズ機能（設定引き出し）
* **テーマ（配色）切り替え**:
  * 「和紙（セピア）」（デフォルト）
  * 「明（ライトモード）」
  * 「暗（ダークモード）」
  * 「漆黒（ブラック）」
* **書体（フォント）切り替え**:
  * 明朝体（読書向けのNoto Serif JP等、適した美しいセリフ系フォント）
  * ゴシック体
* **文字サイズ調整**: 4段階（小、中、大、特大）
* **行間調整**: 3段階（狭い、標準、広い）
* **文字間調整**: 3段階（狭い、標準、広い）

### 3.5 状態保持機能（しおり機能）
* **自動保存（しおり）**: 最後に読んだファイルのスクロール位置（パーセンテージ）を `localStorage`（永続ストレージ）に自動記録。
* **セッション復元**: アプリ再起動時に、前回読んでいたファイル（ファイル名、ファイル種別、コンテンツ内容、スクロール位置）を自動ロード。
* **設定の永続化**: ユーザーが設定したテーマ、フォント、文字サイズ等の表示設定を `localStorage` に保存し、次回起動時に自動適用。

## 4. 非機能要件 (Non-Functional Requirements)

### 4.1 ユーザビリティ・デザイン (UX/UI)
* **高級感のあるデザイン**: デフォルトの和紙風テーマを含め、滑らかなアニメーション、美しく調和のとれた配色（HSL等）、フォントによるプレミアムな読書体験。
* **レスポンシブ対応**: PC大画面では2カラム（見開き）風、モバイル端末では1カラム風の最適なカラム幅（`column-width`）を自動設定し、画面幅に応じてレイアウトが自動的に最適化されること。
* **自動レイアウト再計算**: ウィンドウサイズ変更時や表示設定（文字サイズ等）変更時に、スクロール位置（進捗率基準）を正確に再計算し、読んでいる位置を見失わないようにする。

### 4.2 セキュリティ・安全性
* **エスケープ処理**: テキスト読み込み時にHTMLタグを自動エスケープし、スクリプト実行などのXSS（クロスサイトスクリプティング）脆弱性を完全に排除して安全にレンダリングすること。

### 4.3 パフォーマンス
* **高速ロード**: 外部ライブラリへの依存を最小限にし、軽量なコード設計により静的ファイルとして一瞬で読み込みが完了する。

## 5. 今後の検討事項 (Future Roadmap)
* **複数ファイルの管理 / 本棚機能**: 過去に読み込んだファイルの履歴管理やブックマークの一覧表示。
* **目次ナビゲーション**: 長編小説における章ごとのジャンプ機能。
